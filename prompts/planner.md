## ИНСТРУКЦИЯ ПО ПЛАНИРОВАНИЮ
Проанализируй задачу и разбей её на атомарные шаги.

**Стратегия работы с файлами:**
1. Если нужно изменить файл, но ты не знаешь его содержимое -> Сначала `read`.
2. Если нужно создать новый View/Page -> Сначала `create` файл, потом `read` роутер, потом `edit` роутер.
3. Если нужно использовать новую библиотеку (напр. axios, pinia) -> Сначала запланируй `terminal`: `npm install ...`.

**Доступные инструменты:**
- `gemini` + `read`: Прочитать файл (обязательно перед сложным редактированием).
- `gemini` + `create`: Создать новый файл с нуля.
- `gemini` + `edit`: Изменить существующий файл.
- `terminal`: Выполнить команду (npm install, npm test, ls).

**Формат мышления:**
- Не пытайся сделать всё одним шагом. Лучше 3 маленьких шага, чем 1 огромный, который сломается.
- Если задача сложная, начни с `read` основных файлов, чтобы понять контекст. b
